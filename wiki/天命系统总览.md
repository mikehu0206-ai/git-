## 核心架构

```
人类用户
   ↓
微信链接 / Terminal / 直接对话 → OpenClaw（唯一入口 + 网关 + 安全审计）
                                    ↓
                            Hermes-Agent（Agent 实际运行时）
                                    ↓
        ┌─────────┬─────────┬─────────┬─────────┬─────────┬─────────┐
        线上运营   内容创作   用户运营   线下运营   AI开发    知识管理
         (3)      (2)       (2)       (3)       (7)       (3)
                                    ↓
                    Obsidian 天命图书馆（Nutstore Sync 多端同步）
```

## 系统核心铁律

1. **雅典娜唯一写入原则**：Obsidian 天命图书馆的写入权限永远只属于雅典娜。其他 Agent 只能向莫娜申请临时单次写入权限。
2. **莫娜不执行原则**：莫娜只负责理解需求、生成 BF、任务分配、结果验收和全局调度，不直接执行具体任务。
3. **日志不可篡改原则**：所有系统日志只能追加，不能修改或删除，由雅典娜统一归档。
4. **权限最小化原则**：所有 Agent 默认只拥有最小必要权限，额外权限必须申请且仅限单次任务。
5. **紧急任务直达原则**：`天命紧急：`指令拥有最高优先级，立即中断所有非紧急任务。

> 详见：[[schema/rules|天命系统全局强制规则]]
> 
> **v1.0-rollback (2026-04-30)**：已回退到 v1.0 核心架构，移除企业微信与 Kimi Code CLI 依赖，保留微信链接、MiniMax Token Plan、并发规则与 Agent 模型配置，启用睡眠节能模式。

## 标准指令

| 指令格式 | 用途 |
|---------|------|
| `天命：[任务内容]` | 普通任务提交 |
| `天命紧急：[任务内容]` | 紧急任务，最高优先级 |
| `天命：查看任务状态` | 查看当前进行中任务 |
| `天命：取消当前任务` | 终止当前 session 任务 |

## 技术栈

| 层级 | 组件 | 路径/版本 |
|------|------|----------|
| 网关层 | OpenClaw | v2026.4.24, `ws://127.0.0.1:18789` |
| 运行时 | Hermes | v0.11.0 |
| 模型 | Volcano Ark + MiniMax Token Plan | `ark-code-latest` / `MiniMax-M2.7` |
| 并发规则 | maxWorkers: 3, maxConcurrent: 4 | 多维度并行计算 |
| 知识库 | Obsidian + Nutstore Sync | Vault: `~/Nutstore/天命图书馆-天命系统v1.0` |
| 安全 | macOS Terminal 直接操作 | 本地命令执行 + 目录白名单锁定 |
| 记忆 | shared_memory (SQLite) | `~/.hermes/profiles/athena/memories/` |
| 睡眠模式 | 无任务自动睡眠 | `.sleep-mode` 标志 + 低功耗守护 |

# 快速故障排查指南

> 适用系统：天命系统 v1.0 | 适用环境：macOS | 最后更新：2026-04-28

---

## 新增问题排查

| 问题现象 | 排查步骤 | 解决方案 |
|---------|---------|---------|
| 发现多个 openclaw-gateway 实例 | 1. 检查自守护配置：`launchctl list | grep com.destiny`<br>2. 确认是否同时有 LaunchAgent 和手动启动 | 若仅需单实例，禁用其中一个自启动项，重启系统 |
| Discord 插件连接失败 | 1. 检查网络连接：`ping discord.com`<br>2. 查看 Hermes 日志 | 如不需要 Discord 通知，可在配置中禁用插件 |

---

## 问题速查表

| 问题现象 | 排查步骤 | 解决方案 |
|---------|---------|---------|
| 发送"天命："指令无响应 | 1. 检查 OpenClaw Gateway 是否运行：`ps aux \| grep openclaw`<br>2. 检查 WebSocket 端口是否监听：`lsof -i :18789`<br>3. 检查 Hermes 是否响应：`hermes --version` | 重启 OpenClaw：`pkill openclaw && openclaw serve`（或重新启动 gateway 进程） |
| Agent 执行任务卡住 | 1. 查看 OpenClaw 日志：`tail -50 ~/.openclaw/logs/gateway.log`<br>2. 查看安全审计日志：`tail -20 ~/Documents/天命图书馆/00_系统核心/系统日志/security-audit.log`<br>3. 检查是否有 hook 拦截 | 发送新指令覆盖当前 session，或等待当前任务超时（默认 600 秒） |
| 知识库同步失败 | 1. 检查 Nutstore Sync 插件状态（Obsidian → 设置 → 社区插件 → Nutstore Sync）<br>2. 检查 Vault 路径是否存在：`ls -l ~/Nutstore/天命图书馆-天命系统v1.0/`<br>3. 检查外置盘是否挂载：`ls /Volumes/OpenClawData/` | 重新挂载外置盘，或在 Obsidian 中重新启用 Nutstore Sync 插件 |
| Kimi Code 调用失败 | 1. 检查 API 密钥：`cat ~/.openclaw/openclaw.json \| grep apiKey`<br>2. 检查网络连接：`ping ark.cn-beijing.volces.com`<br>3. 查看 gateway 错误日志：`tail -50 ~/.openclaw/logs/gateway.err.log` | 更新 `~/.openclaw/openclaw.json` 中的 API 密钥配置 |
| 紧急任务无法中断 | 1. 检查当前运行的 OpenClaw session：`ls -lt ~/.openclaw/agents/*/sessions/`<br>2. 发送新的 `--local` 指令强制创建新 session | 如 gateway 无响应，执行 `pkill openclaw` 后重新启动 |
| Agent 写入文件后"看不见" | 1. 检查文件是否在 Vault 中：`find ~/Nutstore/天命图书馆-天命系统v1.0/ -name "*文件名*"`<br>2. 检查是否写入了 workspace：`find ~/.openclaw/workspace* -name "*文件名*"`<br>3. 查看安全审计日志是否有拦截记录 | 如写入 workspace，手动移动到 Vault 正确路径，并通知赛特加固 |

## 关键路径速查

| 组件 | 路径/命令 |
|------|----------|
| OpenClaw 配置 | `~/.openclaw/openclaw.json` |
| OpenClaw 日志 | `~/.openclaw/logs/gateway.log` |
| OpenClaw 错误日志 | `~/.openclaw/logs/gateway.err.log` |
| 安全审计日志 | `~/.openclaw/logs/security-alert.log` |
| Hermes 配置 | `~/.hermes/profiles/athena/config.yaml` |
| Vault（外置盘） | `/Volumes/OpenClawData/我的坚果云/天命图书馆-天命系统v1.0/` |
| Vault（访问入口） | `~/Nutstore/天命图书馆-天命系统v1.0/` |
| LaunchAgent 状态 | `launchctl list \| grep com.destiny` |
| 守护进程日志 | `~/DestinySystem/logs/obsidian-sync.log` |

## 常见误区

1. **不要使用 `systemctl`** —— macOS 没有 systemd，使用 `launchctl` 或 `pkill`/`kill` 管理进程
2. **不要手动修改 agent prompt** —— 所有修改应通过"天命："指令提交给赛特处理
3. **不要把文件写入 `~/.openclaw/workspace/`** —— 这是运行时目录，Obsidian 读不到
4. **不要删除 `.obsidian/nutstore-sync/`** —— 这是同步插件的数据目录

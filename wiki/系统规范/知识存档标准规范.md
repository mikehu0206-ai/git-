# 天命系统知识存档标准规范

> **强制执行**：所有存入天命图书馆的知识存档必须遵循本规范
> **设计目标**：Agent 可快速解析、机器可读、易于搜索、状态可追踪
> **维护者**：雅典娜

---

## 一、文件命名规范

```
{主题描述}-{类型}-结构化存档.md
```

**示例：**
- `吴恩达-Agent智能体教程-结构化存档.md`
- `OpenClaw-MCP协议-结构化存档.md`
- `小红书运营方法论-结构化存档.md`

---

## 二、YAML Frontmatter 头部元数据（必填）

```yaml
---
id: {唯一标识符，格式：{分类}-{名称}-{年份}}
title: {完整标题}
author: {作者/来源}
created_date: {YYYY-MM-DD}
version: {语义化版本号 x.y}
status: {pending | active | archived | deprecated}
tags:
  - {标签1，小写，用-连接}
  - {标签2}
source_url: {来源链接，可选}
related_systems:
  - {关联的系统组件，可选}
---
```

**必填字段说明：**

| 字段 | 格式要求 | 示例 |
|------|---------|------|
| `id` | 全局唯一，小写，用-连接 | `course-andrew-ng-agentic-ai-2026` |
| `title` | 完整可读标题 | `吴恩达 Agentic AI 智能体完整教程` |
| `created_date` | YYYY-MM-DD 格式 | `2026-04-29` |
| `version` | 语义化版本 | `1.0` |
| `status` | 枚举值 | `pending` / `active` / `archived` / `deprecated` |
| `tags` | 数组，小写，用-连接 | `[agent, tool-use, mcp]` |

## 四、模块化内容结构规范

### 4.1 概念/模式/技能 模块

每个独立概念使用独立的 YAML 代码块封装：

```yaml
{模块类型}_id: {唯一标识}
name: {名称}
core_idea: {1句话核心思想}
application_scenarios:
  - {场景1}
  - {场景2}
key_techniques:
  - {关键技术1}
  - {关键技术2}
destiny_implementation: {在天命系统中的对应实现}
status: {pending | in_progress | implemented | optimized}
video_segments: [{模块位置}]
```

### 4.2 学习路径/实施计划 模块

```yaml
phase_id: {阶段标识}
duration: {预计时间}
modules: [{关联模块}]
expected_outcomes:
  - {预期成果1}
  - {预期成果2}
action_items:
  - {具体行动1}
  - {具体行动2}
status: pending
```

### 4.3 索引模块

必须包含搜索关键词索引：

## 五、状态字段规范

所有可变更项目必须包含 `status` 字段，使用以下枚举值：

| 状态值 | 含义 | 适用场景 |
|--------|------|---------|
| `pending` | 待处理/待学习 | 新创建尚未开始 |
| `in_progress` | 进行中 | 正在实施/学习 |
| `implemented` | 已实现 | 功能已落地 |
| `optimized` | 已优化 | 持续改进完成 |
| `active` | 激活中 | 规范/模板生效中 |
| `archived` | 已归档 | 历史存档，不再更新 |
| `deprecated` | 已废弃 | 不建议使用 |

## 六、系统映射表（推荐）

涉及技术/方法的存档，推荐包含与天命系统的映射：

| 外部概念 | 系统组件 | 实现状态 | 对应位置 |
|---------|---------|---------|---------|
| {概念1} | {组件A} | ✅ 已实现 | {文件路径} |
| {概念2} | {组件B} | ⚠️ 部分实现 | {文件路径} |
| {概念3} | {组件C} | ⬜ 待开发 | 待规划 |

**状态图标规范：**
- ✅ = 已完整实现
- ⚠️ = 部分实现/待优化
- ⬜ = 未实现/待开发
- 🔄 = 进行中

## 八、审核标准（雅典娜执行）

存档入库前必须通过以下检查：

✅ **检查项**：
1. YAML Frontmatter 完整且格式正确
2. `id` 全局唯一，无重复
3. 包含核心信息速查区
4. 关键概念使用 YAML 代码块封装
5. 所有可变更项有 `status` 字段
6. 包含搜索关键词索引
7. 有明确的维护者和更新规则
8. 格式整洁，无语法错误

❌ **拒收情况**：
- 纯自然语言大段文本（无结构化）
- 无状态追踪机制
- 无搜索索引
- 格式混乱无法解析
